# 睿嶼文化 RUIYU STUDIO — 部署說明（Zeabur）

純靜態官網（6 頁 HTML + `assets/`）。聯絡表單會跨網域呼叫**現有 booking 後端**的 `/api/contact` 寄信。

```
睿嶼官網（Zeabur 靜態服務）  ──表單 POST──▶  現有 booking 後端（Zeabur）/api/contact ──▶ 寄信
```

---

## A. 上線官網（Zeabur 靜態服務）

Zeabur 會**自動判定為靜態網站**（有 `index.html`、無語言 runtime），用 Caddy 服務，不需任何設定檔。

### 方式一：透過 GitHub（建議，與你其他站一致）
1. 在 GitHub 建一個新的 repo（例如 `ruiyu-website`）。
2. 把本資料夾推上去（git 已初始化並完成首次 commit）：
   ```bash
   cd ~/Desktop/睿嶼
   git remote add origin git@github.com:<你的帳號>/ruiyu-website.git
   git branch -M main
   git push -u origin main
   ```
3. Zeabur → 你的專案 → **Add Service → Deploy from GitHub** → 選 `ruiyu-website`。
4. Zeabur 自動偵測為 Static、部署完成，會給你一個網址如 `https://ruiyu.zeabur.app`。
   （在服務的 **Networking / Domains** 可自訂前綴或綁自訂網域。）

### 方式二：Zeabur CLI（不用 GitHub，直接上傳資料夾）
```bash
cd ~/Desktop/睿嶼
npx zeabur@latest deploy
```
依提示登入、選專案即可。

## B. 讓聯絡表單能真的寄信

表單目前 POST 到 `assets/contact.js` 裡設定的後端位址。上線需三步：

1. **重新部署後端**：本次已在 booking 後端新增 `/api/contact` 路由。到 booking 後端的 git repo：
   ```bash
   cd ~/Desktop/0330預約系統
   git add backend/ && git commit -m "feat: 睿嶼官網聯絡表單 /api/contact"
   git push
   ```
   Zeabur 後端服務會自動重新部署，`/api/contact` 隨即上線。

2. **記下後端公開網址**（Zeabur 後端服務的 Domain），例如 `https://your-backend.zeabur.app`。

3. **設定官網的 API 位址**：編輯 `assets/contact.js` 最上面：
   ```js
   const API_BASE = 'https://your-backend.zeabur.app';   // ← 改成你的後端網址
   ```
   然後 `git commit && git push`（或重新 `zeabur deploy`），官網會自動更新。

4. **（可選、更安全）鎖定來源**：到後端 Zeabur 服務的環境變數加：
   ```
   RUIYU_SITE_ORIGIN=https://ruiyu.zeabur.app
   RUIYU_CONTACT_TO=hello.ruiyustudio@gmail.com
   ```
   不設 `RUIYU_SITE_ORIGIN` 則預設開放所有來源（表單仍可運作）。

> ⚠️ 後端要能寄信，需確認 booking 後端的 SMTP 已設定（你現有的預約提醒信已在運作，即代表 OK）。

## C. 綁自訂網域（買網域之後）

Zeabur 官網服務 → **Domains → Add Domain** → 輸入你的網域 → 依指示到網域商設定 CNAME → Zeabur 自動配 SSL。
