# 睿嶼文化 RUIYU STUDIO — 部署說明（Zeabur）

兩個各自獨立的服務，都放 Zeabur：

```
睿嶼官網（Zeabur 靜態服務）  ──表單 POST──▶  睿嶼 Contact API（Zeabur Node 服務）──▶ 寄信
  這個資料夾                                  ~/Desktop/睿嶼-contact-api
```

> 聯絡表單用的是**專屬的獨立後端**（`睿嶼-contact-api`），與預約系統完全分離。

---

## A. 上線官網（Zeabur 靜態服務）

Zeabur 會**自動判定為靜態網站**（有 `index.html`、無語言 runtime），用 Caddy 服務，不需任何設定檔。

### 方式一：透過 GitHub（建議）
1. GitHub 建 repo（如 `ruiyu-website`），推上去：
   ```bash
   cd ~/Desktop/睿嶼
   git remote add origin git@github.com:<你的帳號>/ruiyu-website.git
   git branch -M main && git push -u origin main
   ```
2. Zeabur → 專案 → **Add Service → Deploy from GitHub** → 選 `ruiyu-website`。
3. 部署完成拿到網址如 `https://ruiyu.zeabur.app`（可在 Domains 綁自訂網域）。

### 方式二：Zeabur CLI（免 GitHub）
```bash
cd ~/Desktop/睿嶼 && npx zeabur@latest deploy
```

---

## B. 上線聯絡表單後端（Zeabur Node 服務）

後端專案在 **`~/Desktop/睿嶼-contact-api`**，部署步驟見該資料夾的 `README.md`，重點：

1. 把 `睿嶼-contact-api` 推到另一個 GitHub repo → Zeabur Add Service → 自動偵測 Node。
2. 在服務 **Variables** 填 SMTP 等環境變數（照 `.env.example`）：
   `SMTP_HOST` `SMTP_PORT` `SMTP_SECURE` `SMTP_USER` `SMTP_PASS`
   `EMAIL_FROM` `EMAIL_FROM_NAME` `RUIYU_CONTACT_TO`
   `RUIYU_SITE_ORIGIN`＝官網網址（例如 `https://ruiyu.zeabur.app`）
3. 產生公開網址（如 `https://ruiyu-api.zeabur.app`），開 `/api/health` 確認回 `{"success":true}`。

> Gmail 需用「應用程式密碼」（Google 帳戶 → 安全性 → 兩步驟驗證 → 應用程式密碼）。

---

## C. 把官網接上後端

拿到後端公開網址後，編輯本資料夾 `assets/contact.js` 最上面：
```js
const API_BASE = 'https://ruiyu-api.zeabur.app';   // ← 改成 Contact API 的網址
```
再 `git commit && git push`（或重新 `zeabur deploy`），官網即更新。

---

## D. 綁自訂網域（買網域之後）
官網服務 → **Domains → Add Domain** → 依指示到網域商設 CNAME → Zeabur 自動配 SSL。
